ARCHITECTURE: BEFORE → AFTER MIGRATION
═══════════════════════════════════════════════════════════

BEFORE (CURRENT - 2 OCR SYSTEMS):
════════════════════════════════════

main.js
  ↓
  ├─→ locator.js (Tesseract.js - Node.js)
  │    ↓
  │    └─→ tesseract.js npm package
  │         └─→ WASM OCR engine
  │
  └─→ locator-pywinauto.js (PyWinAuto - Python)
       ↓
       └─→ python-ipc.js → Flask API (port 5555)
            └─→ ocr_service.py → PyWinAuto (native UI)

PROBLEMS:
• Code duplication
• Two separate OCR systems
• Large bundle size (tesseract.js + WASM)
• Complex conditional logic

═══════════════════════════════════════════════════════════

AFTER (TARGET - 1 UNIFIED OCR SYSTEM):
══════════════════════════════════════

main.js
  ↓
  └─→ locator-pywinauto.js (SINGLE ENTRY POINT)
       ↓
       └─→ python-ipc.js → Flask API (port 5555)
            ↓
            └─→ ocr_service.py
                 ├─→ PyWinAuto (native UI)    [95% accuracy]
                 └─→ pytesseract (OCR)        [70% accuracy]

BENEFITS:
• Single code path
• Unified OCR in Python
• Smaller bundle size
• Simpler logic
• Better performance

═══════════════════════════════════════════════════════════

LOCATION PRIORITY CHAIN:
════════════════════════

User asks: "how do I save a file?"
         ↓
    main.js captures question
         ↓
    AI Vision Model (Gemini/Groq)
    Returns: [{target: "Save", box: [rough coords]}]
         ↓
    locateSteps(steps)  ← ENTRY POINT
         ↓
    ┌────────────────────────────────┐
    │ locator-pywinauto.js           │
    └────────────────────────────────┘
         ↓
    Priority Chain:
    
    1. Try PyWinAuto (native Windows UI)
       └─→ Success? Return coordinates
           └─→ Fail? Continue ↓
    
    2. Try Python OCR (pytesseract)
       └─→ Success? Return coordinates
           └─→ Fail? Continue ↓
    
    3. Use AI box (fallback)
       └─→ Return rough AI coordinates

DATA FLOW EXAMPLE:
══════════════════

1. User: "how do I click save button?"
   
2. main.js → AI Vision → [{target:"Save", box:[500,500,550,600]}]

3. main.js → locateSteps(steps)

4. locator-pywinauto.js → _locateWithPyWinAuto()
   → Python /elements/active-window
   → Finds button with automation_id="btnSave"
   → Returns: {box:[exact coords], confidence:0.95}
   
5. Return to user: Highlight "Save" button with spotlight

─────────────────────────────────────────────────────────────

ALTERNATIVE FLOW (OCR FALLBACK):

1. User: "how do I click search?" (in web browser)

2. main.js → AI Vision → [{target:"Search", box:[100,200,150,300]}]

3. locator-pywinauto.js → _locateWithPyWinAuto()
   → Python /elements/active-window
   → No native element found (web content)
   → Returns: null
   
4. locator-pywinauto.js → _locateWithPythonOCR()
   → Python /ocr/text-locate
   → pytesseract.image_to_data(screenshot)
   → Finds "Search" text in image
   → Returns: {box:[exact coords], confidence:0.75}

5. Return to user: Highlight "Search" text with spotlight
